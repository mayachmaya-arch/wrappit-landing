# Wrappit — עמוד נחיתה שיווקי

עמוד נחיתה סטטי (Vite + React + Tailwind CSS v4), בעברית ו-RTL מלא, בנוי לפי עיצוב Figma.

הקובץ: `jCymW3WjTwLSnlOwawanAT`, נוד `982:5300`.

## הרצה מקומית

דורש Node.js 18 ומעלה.

```bash
npm install
npm run dev       # http://localhost:5173
```

פקודות נוספות:

```bash
npm run build      # בנייה סטטית ל-dist/, מוכן לפריסה לכל שירות (Vercel, Netlify, S3, GitHub Pages...)
npm run preview    # הרצת ה-build המקומי לבדיקה
npm run lint       # oxlint
```

## מבנה הפרויקט

```
src/
  components/
    Header.jsx          נבר עליון: לוגו, ניווט, שתי כפתורי CTA
    Hero.jsx             כותרת מתחלפת (יום הולדת? / בר מצווה? / ...) + תת-כותרת + CTA
    ProblemSection.jsx   "מצד אחד / מצד שני" — כאבי קוני מתנה מול בעלי עסקים
    DarkGallery.jsx       הגלריה הכהה עם תצוגת מוצרים + מוקאפ טלפון
    Marquee.jsx           הבאנר הסגול הנע עם רעיונות למתנה
    PhotoFrame.jsx        עטיפת תמונה עם fallback גרדיאנט למקרה שהתמונה חסרה
  assets/images/          כל התמונות (כרגע placeholder-ים, ראו מטה)
  App.jsx                 הרכבת כל הסקשנים
  index.css                טוקני עיצוב של Tailwind (צבעים, פונטים, אנימציית ה-marquee)
```

## RTL

`index.html` מוגדר `<html lang="he" dir="rtl">` וגם `<body dir="rtl">`, והעטיפה הראשית ב-`App.jsx` מוסיפה `dir="rtl"` מפורש. סדר האלמנטים ב-JSX (למשל לוגו → ניווט → CTA בהדר) נבנה כך שינוצל כיוון ה-flex הטבעי תחת RTL — הלוגו נשאר מימין (התחלה) והכפתורים משמאל (סוף), בדיוק כמו בעיצוב המקורי, בלי טריקים של pixel-positioning.

## אנימציות

- **כותרת מתחלפת (Hero)** — 5 הביטויים ("יום הולדת?", "בר מצווה?" וכו') מתחלפים כל 2.8 שניות עם fade, לפי הלוגיקה מ-Figma (היה מבוסס על ספריית `motion`, כאן מומש עם `useState`/`setInterval` + CSS transitions כדי לא להוסיף תלות מיותרת לעמוד סטטי).
- **הבאנר הנע (Marquee)** — רשימת הרעיונות משוכפלת פעמיים ונעה ברצף אינסופי (`@keyframes marquee-scroll` ב-`index.css`), נטויה ב-2.7deg כמו במקור.

## מה עוד חסר להשלמת העמוד

הסקשנים שנבנו מכסים רק את מה שהתבקש (Header, Hero, בעיה, גלריה כהה, Marquee). **חסרים עדיין הסקשנים הבאים** כדי שהעמוד יהיה שלם:

1. **"הפתרון שלנו"** — כרגע יש רק כותרת מעבר ("פתרון פשוט כל כך") בסוף סקשן הבעיה, בלי תוכן בפועל. יש קישור עוגן `#solution` שכבר מצביע לסקשן הגלריה הכהה כפלייסהולדר זמני.
2. **"ככה זה עובד, הכי פשוט בעולם!"** — יש קישור ניווט `#how-it-works` בהדר, אבל אין סקשן יעד. צריך 3-4 שלבים מוסברים (למשל: בוחרים מתנה → משלמים → מקבל המתנה בוחר בעצמו מה לממש).
3. **פרייסינג** — אין שום התייחסות למודל תמחור (עמלה לעסקים? מנוי? חינם לקונים?).
4. **פוטר** — אין פוטר בכלל: לינקים משפטיים, רשתות חברתיות, פרטי יצירת קשר, ניוזלטר.
5. **טפסי הרשמה בפועל** — כל כפתורי ה-CTA ("הצטרפו כעסק מייסד", "לפתיחת חנות" וכו') הם קישורי `#` בלבד וצריכים להוביל לטופס/דף אמיתי.
6. **SEO / מטא-דאטה מלאים** — יש `<title>` ו-`<meta description>` בסיסיים, אבל חסרים Open Graph, favicon אמיתי (יש רק placeholder), ו-`sitemap`.
7. **נגישות** — כדאי מעבר נוסף על ניגודיות צבעים (טקסט cloud על תמונת הרקע בהירה מדי במקומות), ותיוג ARIA לכפתורי ניווט מתחלפים.

## גופנים

- **Noto Sans Hebrew** (משקלים 300/400/600/700/900) ו-**Berkshire Swash** (ללוגו "Wrappit") נטענים מ-Google Fonts דרך `index.html`.
- כותרת ה-Hero בעיצוב המקורי משתמשת בפונט מותאם **"Reisinger - Michal"** — פונט מסחרי שאינו זמין ב-Google Fonts. כרגע יש נפילה (`fallback`) ל-Noto Sans Hebrew Black שנראה טוב אך שונה מהמקור. אם יש רישיון לפונט, הוסיפו את קובץ הפונט תחת `src/assets/fonts/` והגדירו `@font-face` ב-`src/index.css` תחת המשתנה `--font-display`.
